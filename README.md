# fe-interview-prep

프론트엔드 인터뷰 준비를 체계적으로 진행하기 위한 리포지토리입니다. 주차별 목표와 학습 흐름을 정리하고, 플래시카드·패턴·문제 풀이·회고를 구조화해 반복 학습을 쉽게 만듭니다.

## 활용 가이드

1. **주차별 계획 수립**: 아래 제안 스케줄을 참고해 주차별 목표를 잡고, 각 주의 시작에 개인화된 목표를 명시하세요.
2. **플래시카드 복습**: `flashcards/` 내 파일을 Anki 등 SRS 도구와 함께 사용하며, 틀린 카드 내용은 `retrospectives/`에 이유와 함께 기록합니다.
3. **패턴 연습**: `patterns/`의 구현 예제를 직접 타이핑하며, 변형 케이스를 `questions/`에 추가합니다.
4. **문제 풀이**: 카테고리별로 질문을 추가하고 답안을 작성합니다. 틀린 포인트는 회고에 반영합니다.
5. **회고**: 매주 말에 새 파일을 추가해 실패 사례, 놓친 포인트, 개선 액션을 정리합니다.

## 주차별 제안 목표 (샘플)

| 주 | 목표 | 체크리스트 |
| --- | --- | --- |
| 1주차 | JS 핵심 복습 & 비동기 | 클로저·스코프, 프로토타입, 이벤트 루프, 프라미스/async-await, `flashcards/js-core.md`·`flashcards/async.md` 완독 |
| 2주차 | DOM/브라우저 심화 & 렌더링 | DOM API, 레이아웃/리플로우, 이벤트 위임, 성능 최적화, `flashcards/dom.md` 복습 |
| 3주차 | 패턴 & 코드 구현 감각 | `patterns/` 내 패턴 구현, 변형 문제 추가, `questions/js/`·`questions/ui/` 연습 |
| 4주차 | 시스템 디자인 | `flashcards/system-design.md` 복습, `questions/system-design/` 사례 정리 |
| 5주차 | 종합 리허설 | 모의 인터뷰, 시간 제한 문제 풀이, 빈출 질문 위주 리마인드 |

## 구조

- `flashcards/`: 개념 플래시카드 모음
- `patterns/`: 자주 나오는 구현 패턴 정리
- `questions/`: 카테고리별 예상 질문 & 답변
- `retrospectives/`: 회고와 개선 계획

## 작성 팁

- 각 파일은 짧은 질문/답변 혹은 코드 스니펫 중심으로 유지합니다.
- 틀린 문제는 바로 `retrospectives/`에 링크와 함께 기록해 같은 실수를 줄입니다.
- 실무 경험·프로젝트 사례와 연결 지어 설명을 준비하면 면접 답변이 풍부해집니다.
